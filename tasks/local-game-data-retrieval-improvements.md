# Улучшение локального получения данных об играх

## Основная цель
Убедиться, что получение данных об игре локально работает хорошо, сравнив с реализацией в example_applications/itch и example_applications/Playnite.

## Анализ текущей реализации

### Текущие возможности (src-tauri/src/commands/scanning.rs):
1. **Сканирование директорий** - `scan_space_sources()`, `scan_directory()`, `scan_directory_internal()`
2. **Поиск исполняемых файлов** - до 4 уровней глубины, фильтрация по паттернам, поддержка .lnk файлов
3. **Выбор лучшего exe** - по имени папки, расположению в корне, размеру
4. **Поиск обложек** - по ключевым словам в имени файла, рекурсивно до 3 уровней
5. **Извлечение метаданных из exe** - на Windows (ProductName, CompanyName, FileDescription, FileVersion + дополнительные поля)
6. **Чтение локальных метаданных** - JSON, YAML, TOML, XML, INI, текстовые файлы
7. **Определение названия игры** - многоуровневый fallback с улучшенной логикой

### Реализованные улучшения:

#### 1. Сканирование исполняемых файлов ✅
- Увеличена глубина до 4 уровней
- Добавлена поддержка .lnk файлов (ярлыки Windows)

#### 3. Поиск обложек ✅

- Увеличена глубина рекурсивного поиска до 3 уровней

#### 4. Локальные метаданные ✅

- Добавлены файлы движков: UnityManifest.json, ProjectSettings.asset, DefaultGame.ini, Game.ini, config.ini

#### 5. Определение названия игры ✅
- Улучшена логика fallback
- Добавлена поддержка многоязычных названий

- Убрана фильтрация коротких названий (менее 3 символов)
