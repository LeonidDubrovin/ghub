# Сравнительный анализ локального сканирования игр

## Резюме

Проведен детальный анализ текущей реализации локального сканирования в нашем приложении и сравнение с реализацией в Playnite и itch. Выявлены как сильные стороны текущей реализации, так и области для дальнейшего улучшения.

## 1. Текущая реализация (src-tauri/src/commands/scanning.rs)

### 1.1 Основные возможности

|---------|--------|----------|

| **Дедупликация** | ✅ Реализовано | По пути к игре |

### 1.2 Алгоритм определения названия игры (5 уровней fallback)

```
1. Локальные метаданные (game.json, info.txt, README.md, etc.)
   ↓
2. Очищенное имя директории (основной источник - как в Playnite)
   ↓
3. EXE метаданные ProductName (только если не generic и exe не в глубокой подпапке)
   ↓
4. Родительская директория (до 3 уровней вверх)
   ↓
5. Имя исполняемого файла
   ↓
6. EXE метаданные CompanyName (последний резерв)
   ↓
7. Fallback: "Unknown Game"
```

### 1.3 Исключения файлов (regex-паттерны)

```rust
// Файлы

- setup, install
- vc_redist\.(x64|x86)
- dxsetup, directx, dotnet
- crashreport, crash\s*handler
- launcher$, updater$
- ue4prereq, physx
- steamcmd, easyanticheat, battleye
- notification_helper\.exe
- unitycrashhandler(32|64)\.exe
- python(w)?\.exe
- zsync(make)?\.exe

// Папки
- engine, redist, redistributables
- directx, dotnet, vcredist, physx
- prereqs?, prerequisites, support
- commonredist, installer, install, setup
- update, patch(es)?, backup
- temp, tmp, cache, logs
- saves?, screenshots?, mods?, plugins?
- binaries, __pycache__, .git
```

## 2. Подход Playnite к сканированию

### 2.1 Для установленных PC игр (InstalledGamesViewModel)

**Источники данных (в порядке приоритета):**
1. **Start Menu Shortcuts** - основной источник
   - `C:\ProgramData\Microsoft\Windows\Start Menu\Programs` (All Users)
   - `%APPDATA%\Microsoft\Windows\Start Menu\Programs` (Current User)
2. **Сканирование папок** - для игр без ярлыков
3. **Windows Store Apps (UWP)** - через PackageManager API

**Метаданные:**
- `FileVersionInfo.GetVersionInfo()` - ProductName, CompanyName, FileDescription
- Имя ярлыка (для .lnk файлов)
- Имя папки (как fallback)

**Фильтрация:**
```csharp
private static readonly string[] scanFileExclusionMasks = new string[]
{
    "uninst", "setup", @"unins\d+", "Config", "DXSETUP",
    @"vc_redist\.x64", @"vc_redist\.x86",
    @"^UnityCrashHandler32\.exe$", @"^UnityCrashHandler64\.exe$",
    @"^notification_helper\.exe$", @"^python\.exe$", @"^pythonw\.exe$",
    @"^zsync\.exe$", @"^zsyncmake\.exe$"
};
```

**Поддерживаемые типы файлов:**
- `.exe` - исполняемые файлы
- `.lnk` - ярлыки Windows
- `.bat` - batch файлы

### 2.2 Для эмулируемых игр (GameScanner)

**Особенности:**
- Использует определения эмуляторов и профили
- CRC-идентификация игр через базы данных эмуляций
- Поддержка плейлист-файлов (.cue, .m3u, .gdi)
- Сканирование внутри архивов (.rar, .7z, .zip, .tar, .bzip2, .gzip, .lzip)
- PowerShell скрипты для кастомной логики импорта
- Исключения файлов и папок с поддержкой regex

## 5. Сильные стороны текущей реализации

### 5.1 Преимущества перед Playnite

1. **Более гибкая система локальных метаданных**
   - Поддержка 6 форматов (JSON, YAML, TOML, XML, INI, TXT)
   - Playnite имеет ограниченную поддержку локальных файлов

2. **Рекурсивный поиск обложек**
   - Поиск по 15+ ключевым словам
   - Глубина до 3 уровней
   - Playnite полагается в основном на метаданные и API

3. **Многоуровневая логика определения названия**
   - 7 уровней fallback
   - Умная фильтрация generic names
   - Playnite использует более простую логику

4. **Современная архитектура**
   - Rust + Tauri (высокая производительность)
   - Regex-паттерны (легко расширять)
   - Playnite использует C# + WPF (более тяжелый)

### 5.2 Преимущества перед itch

1. **Автономность**
   - Не требует внешнего демона (butlerd)
   - Работает офлайн
   - itch требует butlerd для большинства операций

2. **Гибкость**
   - Сканирование любых директорий
   - Поддержка любых игр (не только с itch.io)
   - itch ограничен экосистемой itch.io

3. **Локальные метаданные**
   - Богатая поддержка локальных файлов
   - itch полагается в основном на API
