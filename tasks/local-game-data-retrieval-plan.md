# План улучшения локального получения данных об играх

## Анализ текущего состояния

### Уже реализовано (в scanning.rs):
1. ✅ Глубокое сканирование исполняемых файлов (4 уровня)
2. ✅ Поддержка ярлыков Windows (.lnk файлы)
3. ✅ Расширенная фильтрация исполняемых файлов
4. ✅ Расширенные форматы изображений (gif, tiff, tif)
5. ✅ Расширенные ключевые слова для обложек
6. ✅ Расширенные пути поиска обложек
7. ✅ Рекурсивный поиск изображений (2 уровня)
8. ✅ Расширенные форматы метаданных (YAML, TOML, XML, INI)
9. ✅ Расширенное извлечение метаданных из exe
10. ✅ Улучшенное определение generic names

### Проблемы, требующие исправления:

#### 1. Парсинг локальных метаданных
**Проблема**: Функция `read_local_metadata` поддерживает только JSON и текстовые файлы, но METADATA_FILES включает YAML, TOML, XML, INI файлы.
**Решение**: Добавить парсеры для YAML, TOML, XML, INI форматов.

#### 2. Поиск обложек
**Проблема**: Рекурсивный поиск ограничен 2 уровнями глубины.
**Решение**: Увеличить глубину поиска до 3 уровней.

#### 3. Извлечение метаданных из exe
**Проблема**: Дополнительные поля (ProductVersion, LegalCopyright и т.д.) извлекаются, но не сохраняются в структуру ExeMetadata.
**Решение**: Добавить эти поля в структуру ExeMetadata и сохранять их.

#### 4. Определение названия игры
**Проблема**: Логика fallback работает, но можно улучшить анализ названий папок.
**Решение**: Добавить более умный анализ названий папок, включая поддержку многоязычных названий.

#### 5. Кэширование результатов
**Проблема**: Нет кэширования результатов сканирования.
**Решение**: Добавить простое кэширование для ускорения повторных сканирований.

## Детальный план улучшений

### Этап 1: Исправление парсинга локальных метаданных
1. Добавить поддержку YAML файлов (game.yaml, info.yaml, metadata.yaml)
2. Добавить поддержку TOML файлов (game.toml, info.toml, metadata.toml)
3. Добавить поддержку XML файлов (game.xml, info.xml, metadata.xml)
4. Добавить поддержку INI файлов (config.ini, game.ini)

### Этап 2: Улучшение поиска обложек
1. Увеличить глубину рекурсивного поиска до 3 уровней
2. Добавить больше путей поиска (screenshots, promo и т.д.)
3. Улучшить приоритизацию обложек

### Этап 3: Расширение метаданных из exe
1. Добавить поля ProductVersion, LegalCopyright, OriginalFilename, InternalName, Comments в структуру ExeMetadata
2. Сохранять эти поля при извлечении метаданных
3. Использовать эти поля для улучшения определения названия игры

### Этап 4: Улучшение определения названия игры
1. Добавить анализ названий папок с учетом многоязычности
2. Улучшить логику fallback
3. Добавить поддержку файлов конфигурации движков (Unity, Unreal, Godot)

### Этап 5: Добавление кэширования
1. Добавить кэширование результатов сканирования
2. Инвалидация кэша при изменении файлов
3. Оптимизация производительности

## Критерии успеха
1. Находит все исполняемые файлы игр в тестовых директориях
2. Правильно определяет названия игр
3. Находит обложки для большинства игр
4. Извлекает полезные метаданные из exe-файлов
5. Работает стабильно без ошибок
6. Работает быстро (кэширование)

## Тестовые сценарии
1. Игра в глубокой структуре папок (3+ уровней)
2. Игра с несколькими исполняемыми файлами
3. Игра без исполняемого файла (только ярлык)
4. Игра с нестандартными обложками
5. Игра с локальными метаданными в разных форматах (JSON, YAML, TOML, XML, INI)
6. Игра с метаданными в exe-файле
