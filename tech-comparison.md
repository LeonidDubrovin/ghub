# Сравнение технологий для GHub

## 1. Фреймворки для десктопных приложений

### 1.1 Tauri 2.0 (Рекомендуется)

**Язык backend**: Rust  
**Язык frontend**: Any (React/Vue/Svelte/Solid)  
**Размер билда**: 3-15 MB  
**RAM usage**: 30-80 MB

#### Преимущества
- Минимальный размер приложения (нет встроенного Chromium)
- Использует системный WebView (WebView2 на Windows, WebKit на Mac/Linux)
- Rust backend = высокая производительность и безопасность памяти
- Современная архитектура безопасности (sandbox, CSP, capability-based permissions)
- Активное развитие, Tauri 2.0 вышел в 2024
- Deep links, system tray, auto-updater из коробки
- Хорошая документация

#### Недостатки
- Нужно знать Rust (кривая обучения)
- WebView может отличаться на разных платформах
- Меньше готовых библиотек чем у Electron
- Отладка IPC сложнее

#### Когда выбирать
- Нужно лёгкое и быстрое приложение
- Команда готова изучить Rust
- Важна производительность и потребление ресурсов

---

### 1.2 Electron

**Язык backend**: JavaScript/TypeScript (Node.js)  
**Язык frontend**: Any (React/Vue/Angular)  
**Размер билда**: 150-200 MB  
**RAM usage**: 200-400 MB

#### Преимущества
- Огромная экосистема npm
- Много готовых примеров и библиотек
- Знакомый стек для веб-разработчиков
- Проверен временем (VS Code, Discord, Slack)
- Одинаковый рендеринг на всех платформах (Chromium)
- Отличная отладка (Chrome DevTools)

#### Недостатки
- Большой размер приложения (включает Chromium)
- Высокое потребление памяти
- Возможные проблемы с производительностью
- Проблемы безопасности (если не настроить правильно)

#### Когда выбирать
- Команда не знает Rust
- Нужен быстрый прототип
- Критично одинаковое поведение UI на всех платформах

---

### 1.3 Avalonia UI (.NET)

**Язык**: C#  
**UI framework**: XAML-подобный  
**Размер билда**: 30-50 MB (с self-contained .NET)  
**RAM usage**: 80-150 MB

#### Преимущества
- Знакомый стек для .NET разработчиков
- MVVM pattern с ReactiveUI
- Хорошая производительность (нативный рендеринг)
- Зрелая экосистема .NET
- Hot reload для UI
- Можно использовать существующий C# код

#### Недостатки
- Меньше сообщество чем у Electron/Tauri
- Меньше готовых UI компонентов
- XAML может быть непривычен веб-разработчикам
- Кроссплатформенность требует тестирования

#### Когда выбирать
- Команда знает C#/.NET
- Есть существующий код на C#
- Нужен нативный look and feel

---

### 1.4 Flutter Desktop

**Язык**: Dart  
**UI framework**: Flutter widgets  
**Размер билда**: 20-40 MB  
**RAM usage**: 100-200 MB

#### Преимущества
- Единый код для desktop/mobile/web
- Отличный UI toolkit
- Hot reload
- Активное развитие от Google

#### Недостатки
- Нужно учить Dart
- Desktop всё ещё относительно новый
- Меньше нативных интеграций
- Ограниченный доступ к системным API

#### Когда выбирать
- Планируется мобильная версия
- Нравится подход Flutter к UI

## 2. Базы данных

### 2.1 SQLite (Рекомендуется)

#### Преимущества
- Один файл = легко бэкапить и переносить
- ACID транзакции
- FTS5 для полнотекстового поиска
- WAL mode для конкурентного доступа
- Нулевая конфигурация
- Проверен десятилетиями

#### Недостатки  
- Не подходит для много-серверных архитектур
- Нет встроенного шифрования

#### Rust библиотеки
- `rusqlite` - низкоуровневый доступ
- `sqlx` - async, compile-time проверка запросов
- `diesel` - ORM

### 2.2 LiteDB

Альтернатива для .NET - документо-ориентированная БД.

#### Преимущества
- Нативная интеграция с C#
- BSON формат
- Простой API

#### Недостатки
- Нет FTS из коробки
- Только для .NET

### 2.3 IndexedDB / localStorage

Для pure-Electron подхода.

#### Недостатки
- Ограничения по размеру
- Нет SQL
- Сложнее бэкапить

## 4. Рекомендуемое решение для GHub

### Основной выбор: **Tauri 2.0 + React + SQLite**

**Причины:**

1. **Производительность** - важна для лаунчера, который работает в фоне
2. **Размер** - пользователи не хотят устанавливать 200 MB для простого лаунчера
3. **Кроссплатформенность** - требуется Windows + Mac + Linux
4. **Heartbeat система** - Rust отлично подходит для фоновых задач
5. **Безопасность** - работа с токенами авторизации требует надёжности
6. **Будущее** - Tauri активно развивается, поддержка Tauri Mobile

### Альтернатива: **Electron + React + better-sqlite3**

Выбирайте если:
- Команда не знает Rust и нет времени на изучение
- Нужен быстрый MVP
- Размер приложения не критичен

### Альтернатива: **Avalonia + C#**

Выбирайте если:
- Команда имеет опыт в .NET
- Нужен максимально нативный UI на Windows

## 5. Оценка трудозатрат

| Вариант | Время до MVP | Сложность поддержки |
|---------|--------------|---------------------|
| Tauri + React | 3-4 месяца | Средняя |
| Electron + React | 2-3 месяца | Низкая |
| Avalonia + C# | 3-4 месяца | Средняя |

MVP включает:
- Базовый UI с библиотекой игр
- Сканирование папок
- Запуск игр
- Учёт времени (без heartbeat)

Полный функционал (с itch.io, heartbeat, scraping): +2-3 месяца
