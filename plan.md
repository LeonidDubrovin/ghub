# GHub - План разработки игрового лаунчера

## 1. Краткий обзор (цель проекта)

Десктопный лаунчер игр (приоритет — Windows, сборки также для macOS и Linux), который:

- Агрегирует игры из разных источников (локальные папки/диски, Steam manifests, itch.io с авторизацией и загрузкой через butler, другие источники)
- Хранит единый каталог метаданных в SQLite с полнотекстовым поиском
- Поддерживает логические **пространства (Spaces)** — соответствующие физическим дискам или библиотекам
- Умеет массово загружать игры из itch.io в выбранные каталоги через Butler
- Надёжно учитывает время игры в реальном времени (heartbeat) с минимальной потерей при аварийном выключении
- Расширяем плагинами для импорта / скрейпинга / способов запуска

## 3. Технические/нефункциональные требования

| Требование | Описание |
|------------|----------|
| **Платформы** | Windows (первичная), macOS, Linux |
| **Offline-first** | Всё работает без интернета; сетевые действия опциональны |
| **Поиск** | Быстрый полнотекстовый поиск (SQLite FTS5, <100ms на 10K игр) |
| **Надёжность** | ACID для метаданных (SQLite WAL mode) |
| **Производительность** | RAM < 100 MB в idle, запуск < 3 сек |
| **Размер приложения** | < 30 MB (без Butler) |
| **Безопасность** | Токены зашифрованы (DPAPI / Keychain / libsecret) |
| **Расширяемость** | Плагинная система (изолированные плагины) |
| **Бэкап** | Один файл БД + папка cache — легко скопировать |

## 5. Архитектура хранения данных

### 5.1 Файловая структура

```
%APPDATA%/GHub/
├── config/
│   ├── settings.json           # Настройки приложения
│   └── spaces.json             # Конфигурация пространств
├── data/
│   ├── ghub.db                 # SQLite база данных
│   ├── ghub.db-wal             # Write-ahead log
│   └── ghub.db-shm             # Shared memory
├── cache/
│   ├── artwork/                # Обложки (SHA256.jpg/webp)
│   └── thumbnails/             # Миниатюры
├── logs/
│   ├── app.log                 # Логи приложения
│   └── playtime.log            # Бэкап сессий
├── plugins/                    # Пользовательские плагины
├── backups/                    # Автобэкапы БД
└── butler/                     # Butler CLI (опционально)
```

### 5.2 Основные таблицы БД

- **spaces** — пространства/библиотеки (id, name, path, type, color)
- **games** — каталог игр (id, title, description, cover, total_playtime)
- **installs** — установки игр (game_id, space_id, path, executable)
- **game_sources** — связь с внешними ID (steam_id, itch_id, igdb_id)
- **tags** / **game_tags** — система тегов
- **play_sessions** — история сессий
- **active_sessions** — heartbeat таблица
- **downloads** — очередь загрузок
- **wishlist** — отложенные ссылки
- **settings** — настройки приложения
- **auth_tokens** — зашифрованные токены

### 5.3 Full-Text Search

```sql
CREATE VIRTUAL TABLE games_fts USING fts5(
    title, description, developer, publisher,
    content='games', content_rowid='rowid'
);
```

## 6. Концепция пространств (Spaces)

### 6.1 Типы пространств

| Тип | Описание | Пример |
|-----|----------|--------|
| **local** | Физическая папка с играми | D:\Games, E:\Indies |
| **steam** | Библиотека Steam | C:\Program Files\Steam\steamapps |
| **itch** | Папка для загрузок itch.io | D:\Games\itch |
| **virtual** | Логическая группировка | "Инди-игры", "Для детей" |

### 6.2 Связь Games ↔ Spaces ↔ Installs

```
┌─────────────────────┐
│       GAME          │  Одна игра (Celeste)
│  (метаданные)       │
└─────────┬───────────┘
          │ 1:N
          ▼
┌─────────────────────┐     ┌─────────────────────┐
│      INSTALL        │     │      INSTALL        │
│  space: "Steam"     │     │  space: "D:\Games"  │
│  path: .../Celeste  │     │  path: D:\...\Cel   │
└─────────────────────┘     └─────────────────────┘
```

Одна игра может быть установлена в нескольких пространствах (например, Steam и локальная копия).

## 7. Учёт времени игры (Heartbeat)

### 7.1 Алгоритм

1. **Запуск игры** → создаём `play_session` (status='active') и `active_session`
2. **Каждые 15 сек** → проверяем процесс, увеличиваем `accumulated_seconds`
3. **Каждые 60 сек** → checkpoint: пишем в `play_sessions`, обновляем `games.total_playtime`
4. **Завершение игры** → финальный checkpoint, status='completed'
5. **При старте приложения** → проверяем `active_sessions`, восстанавливаем незавершённые

### 7.2 Гарантии

- Максимальная потеря при внезапном выключении: **~15 секунд**
- WAL mode SQLite обеспечивает durability
- Логи сессий как дополнительный бэкап

## 8. Интеграция с itch.io

### 8.1 OAuth2 Flow

```
1. User clicks "Login to itch.io"
2. Open browser: https://itch.io/user/oauth?client_id=XXX&redirect_uri=ghub://oauth
3. User authorizes
4. itch.io redirects to ghub://oauth?code=XXX
5. Tauri handles deep link, exchanges code for access_token
6. Token encrypted and stored in auth_tokens table
```

### 8.2 Butler Integration

Butler — официальная утилита itch.io для загрузки игр:

```bash
butler upgrade --json "itch.io/game/channel" "D:/Games/itch/GameName"
```

GHub оркеструет Butler-процессы, парсит JSON progress events.

## 10. Безопасность

- **Токены**: зашифрованы через OS API (DPAPI, Keychain, libsecret)
- **IPC**: Tauri sandbox, whitelist команд
- **Плагины**: изолированный WASM runtime
- **Запуск exe**: только из доверенных путей (в рамках spaces)
